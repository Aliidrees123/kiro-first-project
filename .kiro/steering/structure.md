## Conventions

- Keep `first_project.py` as the main entry point
- As the project grows, organize code into modules/packages under descriptive directory names (e.g., `src/`, or feature-based packages)
- Tests should live in a `tests/` directory at the project root
- Update this file as the structure evolves
